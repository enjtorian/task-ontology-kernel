# Changelog

All notable changes to the Task Ontology Kernel (TOK) project will be documented in this file.

## [1.0.0] - 2026-03-03

### 📚 `docs/` (Whitepaper & Guides)

*   **Core Documentation**:
    *   Authored comprehensive **Whitepaper** (English & Traditional Chinese) defining the Task Ontology Kernel (TOK) — the ontological structure of tasks as AI-native system primitives.
    *   Formalized the **Task Object four-layer structure**: Intent, Context, Strategy, and Evaluation.
    *   Defined **TOK Core Schema** in YAML for standardized task definition, execution contracts, and lifecycle management.
*   **TOCA Architecture**:
    *   Introduced **Task-Oriented Cognitive Architecture (TOCA)** — a closed-loop cognitive system: Capture → Dispatch → Execute → Validate → Evolve.
    *   Established comparison models between TOCA and traditional Chat-based / Tool-based architectures.
*   **Five-Layer Conceptual Stack**:
    *   Defined the complete hierarchy: Paradigm (POG) → Ontology (TOK) → Architecture (TOCA) → Primitive (POG Task) → System (POG OS).
*   **Paradigm Evolution**:
    *   Documented the four eras of software engineering production primitives: Machine-native → Code-native → Service-native → Task-native.
    *   Articulated the shift from code as "core asset" to code as "derivative tool."
*   **Comparative Analysis**:
    *   Four-dimension coverage comparison (Ontology + Execution + AI-native + Versioned) against Temporal, Airflow, Palantir Foundry, Terraform, LangChain, and AutoGen.
    *   Detailed differentiation between TOK (Intent Ontology) and Palantir (Object Ontology).
*   **Guides & References**:
    *   Created **Quick Start** guide for immediate onboarding with TOK concepts.
    *   Added **FAQ** covering general, architecture, and technical questions.
    *   Developed **Glossary** with comprehensive definitions of all TOK/TOCA/POG terminology.
*   **Localization**:
    *   Translated all documentation (Whitepaper, Quick Start, FAQ, Glossary) to **Traditional Chinese (zh-tw)**.
*   **Website**:
    *   Launched documentation site with MkDocs Material theme, deployed via GitHub Pages.
    *   Configured bilingual navigation (English / 繁體中文).
